🔌 API（Application Programming Interface）の基本

API（Application Programming Interface）とは、異なるソフトウェア同士がやり取りするための仕組みのことです。 あなたも普段、知らず知らずのうちにAPIを使っているかもしれません。

たとえば、ウェブサイトで「Googleアカウントでログイン」というボタンを見たことはありませんか？ これは、そのウェブサイトがGoogleのAPIを使って、あなたのアカウント情報を取得しているのです。 APIを使うことで、別々のサービス同士が連携して、便利な機能を提供できるようになります。

💡 簡単に言うと

APIは、別のサービスの機能を借りて使う窓口のようなものです。 これにより、あなたのGASプログラムから、ChatGPTやGemini、Slackなど、外部サービスの機能を呼び出して使うことができるようになります。

🌐 APIの具体例

  • • ChatGPT API：GASからChatGPTに質問を送り、回答を受け取る
  • • Gemini API：GASからGeminiに画像を送り、画像の説明を受け取る
  • • Slack API：GASからSlackにメッセージを送信する
  • • Twitter API：GASからTwitterに自動投稿する

APIの仕組みを、レストランに例えて理解してみましょう。 この例を使うと、APIがどのように動いているのか、イメージしやすくなると思います。

📋 APIの流れ

1

注文を伝える

あなた（GAS）がウェイター（API）に注文を伝えます。
例：「ChatGPTさん、このメール文章を校正してください」

2

キッチンに伝える

ウェイター（API）がキッチン（外部サービス）に注文を伝えます。
APIがリクエストを外部サービスに送信します

3

料理を作る

キッチン（外部サービス）が料理を作ります。
ChatGPTがメール文章を校正します

4

料理を届ける

ウェイター（API）が料理をあなた（GAS）に届けます。
APIが校正後のメール文章を返します

🎯 ポイント

このように、APIはあなたと外部サービスの間を取り持つ役割を果たします。 あなたは直接キッチン（外部サービス）に行く必要はなく、ウェイター（API）に頼むだけで済むのです。

APIを使えば、GASから簡単に外部サービスの機能を呼び出せるようになります。 これにより、GASの可能性が大きく広がり、より高度な自動化が実現できるようになるでしょう。
